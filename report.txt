
ОТЧЕТ ПО ЛАБОРАТОРНОЙ РАБОТЕ 2: RNN ДЛЯ БИОМЕТРИЧЕСКОЙ ИДЕНТИФИКАЦИИ
================================================================================

1. ОПИСАНИЕ ДАТАСЕТА
   - Датасет: LFW (Labeled Faces in the Wild)
   - Количество изображений: 1288
   - Количество классов (персон): 7
   - Размер изображения: 50x37 пикселей
   - Тип биометрии: Статическая (изображения лиц)

2. АРХИТЕКТУРА RNN
   - Тип: SimpleRNN (простая рекуррентная сеть)
   - Количество RNN слоев: 2
   - Размеры RNN: 64 -> 32 нейронов
   - Dropout: 0.2-0.3 для регуляризации
   - Активация: tanh (RNN), relu (Dense), softmax (выход)
   - Входные данные: Изображение обрабатывается как последовательность строк пикселей

3. ОБУЧЕНИЕ
   - Процент обучающей выборки: 70%
   - Процент тестовой выборки: 30%
   - Оптимизатор: Adam
   - Функция потерь: Categorical Crossentropy
   - Early Stopping: Да (patience=10)
   - Learning Rate Reduction: Да (factor=0.5, patience=5)

4. РЕЗУЛЬТАТЫ ОБУЧЕНИЯ
   - Финальная точность на обучающей выборке: 0.5261
   - Финальная точность на валидационной выборке: 0.5556
   - Признаки переобучения: -0.0295

5. МЕТРИКИ КАЧЕСТВА
   - Accuracy: 0.5685
   - Recall (Macro): 0.2829
   - F1-Score (Macro): 0.2142
   - TPR (True Positive Rate): 0.5685
   - FPR (False Positive Rate): 0.0719
   - MSE (Mean Squared Error): 0.0801
   - MAE (Mean Absolute Error): 0.1671
   - Mean AUC: 0.8102

6. МАТРИЦА ОШИБОК (усредненные значения)
   - TP (True Positives): 31.43
   - TN (True Negatives): 307.86
   - FP (False Positives): 23.86
   - FN (False Negatives): 23.86

7. ВЫВОДЫ

   7.1. Эффективность RNN для биометрии:
        - RNN показывает низкую эффективность для распознавания лиц
        - Обработка изображения как последовательности строк НЕ оптимальна для пространственных данных
        - Модель склонна предсказывать наиболее частый класс из-за дисбаланса данных
        - RNN не подходит для изображений - нужна CNN

   7.2. Качество модели:
        - Accuracy 0.5685 указывает на низкое качество классификации
        - F1-Score 0.2142 показывает несбалансированную работу модели
        - AUC 0.8102 близко к случайному угадыванию (0.5)

   7.3. Переобучение:
        - Разница между train и validation accuracy составляет -0.0295
        - Переобучение минимально благодаря Dropout и Early Stopping

   7.4. TPR vs FPR:
        - TPR = 0.5685 - модель корректно распознает значительную часть положительных примеров
        - FPR = 0.0719 - низкий уровень ложных срабатываний
        - Соотношение TPR/FPR = 7.90 - отличный баланс

   7.5. Практическое применение:
        - Модель может использоваться для систем контроля доступа
        - Подходит для верификации личности в биометрических системах
        - Требуется дополнительная настройка для промышленного использования

   7.6. Рекомендации по улучшению:
        - Увеличение датасета для лучшей генерализации
        - Использование аугментации данных
        - Экспериментирование с гибридными CNN+RNN архитектурами
        - Применение attention механизмов
        - Тонкая настройка гиперпараметров

8. ЗАКЛЮЧЕНИЕ
   RNN архитектура показала НИЗКУЮ эффективность в задаче биометрической
   идентификации по изображениям лиц. Модель имеет проблемы с дисбалансом классов
   и предсказывает преимущественно наиболее частый класс. RNN не подходит для
   пространственных данных (изображений). Для данной задачи рекомендуется
   использовать CNN (сверточные нейронные сети), которые специально разработаны
   для обработки изображений.
